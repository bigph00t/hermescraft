# HermesCraft — Project Context

## Overview
20v20 AI Minecraft showdown (Claude vs Codex). Each agent is isolated with its own identity, memory, and plays autonomously.

## Architecture
- **MC Server**: Fabric 1.21.11, Docker on Glass (port 25565)
- **HeadlessMC Client**: Runs HermesBridge mod headlessly on Glass (port 3001)
- **HermesBridge Mod**: Client-side Fabric mod, HTTP API for agent control. Auto-connects via `HERMESCRAFT_SERVER` env var.
- **Agent**: Node.js (agent/index.js), calls HermesBridge HTTP API on port 3001
- **LLM**: Claude OAuth (from Claude CLI on Glass), using OpenAI-compatible endpoint at api.anthropic.com/v1
- **External Access**: mc.alexanderknigge.com (Cloudflare A record → Glass public IP 67.170.228.80, port forwarded 25565→10.0.0.177)

## Build System
- Fabric Loom 1.15.5 requires Gradle 9.2.0 (NOT 8.x — plugin API version mismatch)
- buildscript {} block required (plugins {} won't resolve because .gradle.plugin marker artifact missing from Fabric Maven)
- Yarn mappings: 1.21.11+build.1:v2
- Fabric API: 0.141.3+1.21.11
- Fabric Loader: 0.18.4

## 1.21.11 API Changes (from 1.21.1 port)
- PlayerInventory.selectedSlot → private (use access widener)
- Recipe.getResult(registryManager) → Recipe.getResult() (no args)
- RecipeManager.values() → removed (use getAll(RecipeType))
- Recipe.getIngredients() → returns List<Optional<Ingredient>> not List<Ingredient>
- Ingredient.getMatchingStacks() → Ingredient.getMatchingItems() returns Stream<RegistryEntry<Item>>
- NetworkHandler.sendCommand() → sendChatCommand()
- PlayerInventory.armor.get(i) → player.getEquippedStack(EquipmentSlot)
- Client-side recipe API completely removed — crafting is stubbed in HermesBridge

## Known Issues
- No Baritone for 1.21.11 (latest v1.11.2 only supports 1.21/1.21.1) — pathfinding disabled
- Crafting disabled (client recipe API removed in 1.21.11)
- HeadlessMC particle crash on 1.21.11 (null sprites) — resolved by disabling dummy assets (hmc.assets.dummy=false)
- hmc-specifics mod causes JLine crash in systemd — disabled
- hmc-optimizations not available for 1.21.11 yet

## Commands
Start agent: `ssh glass "cd /opt/hermescraft/agent && source ~/.nvm/nvm.sh && export \$(cat .env | grep -v '^#' | xargs) && node agent/index.js"`
Restart MC client: `ssh glass "sudo systemctl restart hermes-mc-client"`
Restart MC server: `ssh glass "docker restart mc-server"`
Check API: `ssh glass "curl -s http://localhost:3001/state"`
