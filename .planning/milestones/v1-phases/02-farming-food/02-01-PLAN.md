Create `agent/farming.js` following the exact same module-state pattern as builder.js (_activeFarm, start/resume/progress/cancel lifecycle). The farm tool is a sustained action that orchestrates existing mod actions in sequence.

**farming.js implementation:**

Module state: `_activeFarm = null` with fields:
- `origin: {x,y,z}` — front-left corner of the 9x9 farm area
- `phase: 'tilling' | 'planting' | 'waiting' | 'harvesting'`
- `queue: [{x,y,z,action}]` — remaining blocks to process in current phase
- `startedAt: Date.now()`
- `cropType: 'wheat_seeds' | 'beetroot_seeds' | 'carrot' | 'potato'` (default wheat_seeds)
- `stats: { tilled: 0, planted: 0, harvested: 0 }`

**Exports:**

`startFarm(originX, originY, originZ, cropType = 'wheat_seeds')`:
- Validates crop type is one of: wheat_seeds, beetroot_seeds, carrot, potato
- Generates the tilling queue: 9x9 grid EXCLUDING the water channel column (x offset = 4 from origin). That's 72 farmland positions (8 columns x 9 rows).
- Each queue entry: `{ x: originX + col, y: originY, z: originZ + row }` where col skips index 4
- Sets phase to 'tilling'
- Returns `{ success: true, message: "Started farming. 72 blocks to till.", totalBlocks: 72 }`

`async resumeFarm(inventory)`:
- Called each tick when farm is active
- **Tilling phase**: Process up to 2 blocks per tick. For each: call executeAction with `{ type: 'equip', item: 'wooden_hoe' }` (or stone_hoe, iron_hoe — find first hoe in inventory), then `{ type: 'interact_block', x, y, z }` to till the dirt. Remove from queue on success. When queue empty, switch to 'planting' phase and rebuild queue with same positions.
- **Planting phase**: Process up to 3 blocks per tick. For each: call executeAction with `{ type: 'place', item: cropType, x, y: y+1, z }` (crops go one block above farmland at y+1). Note: place at y+1 since farmland is at y and crops sit on top. Actually MC places crops ON farmland — use `{ type: 'place', item: cropType, x, y, z }` with the farmland coordinates since the place handler targets above the block. Remove from queue on success. When queue empty, switch to 'waiting'.
- **Waiting phase**: Return status "Crops growing. Check back later." Set _activeFarm to null (farm cycle complete for now — agent can do other things while crops grow).
- **Harvesting phase** (started by harvestFarm): Process up to 3 blocks per tick. For each mature crop position: call executeAction `{ type: 'look_at_block', x, y: y+1, z }` then `{ type: 'break_block' }`. Remove from queue on success. When done, return `{ complete: true, harvested: stats.harvested }` and set _activeFarm to null.
- If hoe not in inventory during tilling, pause with missingItems: ['wooden_hoe'].
- If seeds not in inventory during planting, pause with missingItems: [cropType].

`startHarvest(originX, originY, originZ)`:
- Creates harvest queue for the same 72 positions (y+1 for crop blocks)
- Sets phase to 'harvesting'
- Returns `{ success: true, message: "Harvesting crops." }`

`getFarmProgress()`:
- Returns string like: "FARMING: tilling (15/72 blocks)" or "FARMING: planting wheat_seeds (30/72)" or "Crops planted. Growing..." or "HARVESTING: 20/72 crops collected"
- Returns empty string if no active farm

`cancelFarm()`, `isFarmActive()` — same pattern as builder.js

**Helper function:**
`findHoe(inventory)` — scans inventory for any item containing 'hoe', returns item ID or null.

**Critical implementation notes:**
- Use `import { executeAction } from './actions.js'` for calling mod actions
- Use `import { fetchState } from './state.js'` if proximity check needed
- The MOD_URL is NOT needed directly — executeAction handles it
- Rate limit: 2 blocks per tick for tilling (slower because equip+interact), 3 for planting/harvesting
- Don't implement growth detection — the LLM will decide when to harvest based on time and visual observation

```markdown
## Cooking
You can cook raw food using `smelt`. Place raw food and fuel in a nearby furnace:
- raw_beef -> cooked_beef (best food, restores 8 hunger)
- raw_porkchop -> cooked_porkchop (restores 8 hunger)
- raw_chicken -> cooked_chicken (restores 6 hunger)
- raw_mutton -> cooked_mutton (restores 6 hunger)
- raw_rabbit -> cooked_rabbit (restores 5 hunger)
- raw_cod -> cooked_cod (restores 5 hunger)
- raw_salmon -> cooked_salmon (restores 6 hunger)
- potato -> baked_potato (restores 5 hunger)
- kelp -> dried_kelp (restores 1 hunger)

Always cook meat before eating — raw chicken can poison you.

## Crop Farming
You have a `farm` tool for automated crop farming:
1. Build a crop-farm blueprint first (provides water + farmland layout)
2. Call: farm(x=..., y=..., z=...) with the farm's origin coordinates
3. Farming tills the soil and plants seeds automatically
4. Crops take ~5-10 minutes real time to grow
5. When crops look mature (tall, golden wheat), call: harvest(x=..., y=..., z=...)

### Crop Types
- wheat_seeds: Plant on farmland. Harvests wheat (for bread) + more seeds
- beetroot_seeds: Plant on farmland. Harvests beetroot
- carrot: Plant carrot directly on farmland
- potato: Plant potato directly on farmland

### Tips
- Farm near water — crops need water within 4 blocks to grow
- Don't walk on farmland — it turns back to dirt
- Craft bread from 3 wheat (no furnace needed)
- Seeds drop from breaking tall_grass

## Food Priority
When hungry, eat in this order:
1. Cooked meat (beef, porkchop — best saturation)
2. Bread (3 wheat, no cooking needed)
3. Baked potatoes, cooked fish
4. Raw vegetables (carrots, beetroot)
5. Raw meat (last resort — raw chicken can poison)

## Animal Breeding Food
- Cows/sheep: wheat
- Chickens: seeds (wheat_seeds, melon_seeds, etc.)
- Pigs: carrots, potatoes, beetroot
- Rabbits: carrots, dandelions
```

```javascript
// FARM-05: Auto-replant saplings after mining logs
if (actionType === 'mine' && result && result.success) {
  const blockName = action.blockName || '';
  const logTypes = ['oak_log', 'birch_log', 'spruce_log', 'jungle_log', 'acacia_log', 'dark_oak_log', 'cherry_log', 'mangrove_log'];
  if (logTypes.some(log => blockName.includes(log.replace('_log', '')))) {
    // Determine matching sapling type
    const saplingMap = {
      'oak': 'oak_sapling', 'birch': 'birch_sapling', 'spruce': 'spruce_sapling',
      'jungle': 'jungle_sapling', 'acacia': 'acacia_sapling', 'dark_oak': 'dark_oak_sapling',
      'cherry': 'cherry_sapling', 'mangrove': 'mangrove_propagule',
    };
    const woodType = Object.keys(saplingMap).find(t => blockName.includes(t));
    if (woodType) {
      const sapling = saplingMap[woodType];
      // Check if agent has the sapling in inventory
      const hasSapling = state.inventory?.some(i =>
        (i.item || i.name || '').replace('minecraft:', '').includes(sapling)
      );
      if (hasSapling) {
        // Best-effort replant — don't block if it fails
        try {
          const pos = state.position;
          if (pos) {
            await executeAction({ type: 'place', item: sapling });
            logInfo(`Auto-replanted ${sapling}`);
          }
        } catch {}
      }
    }
  }
}
```

This is a best-effort hook — it tries to place a sapling at the agent's current position after mining a log. If the agent doesn't have a sapling or the placement fails, it silently continues. The agent will naturally pick up saplings from broken leaves.
