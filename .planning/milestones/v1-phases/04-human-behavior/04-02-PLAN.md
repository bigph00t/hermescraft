Add behavior-aware prompt context and idle/social behavior hints to the action loop. Plan 01 made the planner write behavior mode and needs into plan-context.txt. This plan makes the action loop's prompts explicitly guide the LLM toward idle behaviors (look around, wander, organize) when nothing urgent is happening, and toward socialization (chat about the day, share stories) during night social mode.

Purpose: The constant-purposeful-action pattern is the biggest giveaway that an agent isn't human. Agents need downtime behaviors and social engagement at night to feel real.

   ```javascript

   });
   ```

5. **Idle tick tracking — after the LLM response is processed and action is dispatched** (near the end of tick, after executeAction):
   Track whether the last action was "meaningful" (not just looking around or chatting):
   ```javascript
   // Idle detection — if the last action was info-only or failed, increment idle counter
   const meaningfulActions = new Set(['mine', 'navigate', 'craft', 'place', 'build', 'farm', 'harvest', 'equip', 'break_block', 'fish', 'breed', 'interact_entity', 'interact_block'])
   const lastActionType = actionHistory.length > 0 ? actionHistory[actionHistory.length - 1]?.type : null
   if (lastActionType && meaningfulActions.has(lastActionType) && actionHistory[actionHistory.length - 1]?.success) {
     idleTicks = 0
   } else {
     idleTicks++
   }
   ```

6. **Build idle hint for prompt injection** — compute BEFORE the buildUserMessage call:
   ```javascript
   let idleHint = ''
   if (idleTicks >= 5 && behaviorMode === 'work') {

   } else if (idleTicks >= 3 && behaviorMode === 'social') {
     idleHint = "You're in social mode but haven't been chatting. If there are players nearby, say something — talk about your day, ask what they've been up to, share a story."
   }
   ```

7. **Pass idleHint to buildUserMessage** — add `idleHint,` to the options object in the existing buildUserMessage call.

8. **Reset idle counter on conversation clear** — in the clearConversation calls that already exist (stuck recovery, death, phase transition), also reset: `idleTicks = 0`
