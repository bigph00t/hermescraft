  <name>Task 1: Create cooperation.js — chat-based coordination awareness</name>
  <files>agent/cooperation.js</files>
  <read_first>agent/chat-history.js, agent/social.js</read_first>
  <action>
Create `agent/cooperation.js` with these exports:

1. `parseOtherActivities(chats, selfName)` — Scans recent chat messages (from getRecentChats) for activity announcements. Returns array of `{ player: string, activity: string, raw: string }`. Detection keywords:
   - Mining/gathering: "mining", "getting", "gathering", "chopping", "digging", "collecting"
   - Building: "building", "constructing", "making a", "working on"
   - Farming: "farming", "planting", "harvesting", "fishing"
   - Exploring: "exploring", "checking out", "heading to", "going to"
   - Filter out messages from `selfName` (the agent's own name)
   - Only consider messages from last 5 minutes (compare timestamps to Date.now())

2. `parseResourceNeeds(chats, selfName)` — Scans for resource need mentions. Returns array of `{ player: string, item: string, raw: string }`. Detection patterns:
   - "need(s) X", "looking for X", "anyone have X", "out of X", "running low on X"
   - X is captured as 1-3 words after the keyword phrase
   - Filter out self

3. `parseBuildProjects(chats, selfName)` — Scans for collective build discussions. Returns array of `{ player: string, project: string, location: string|null, raw: string }`. Detection patterns:
   - "let's build", "should we build", "building a X", "started a X"
   - Location extraction: look for coordinate patterns like "at X,Y,Z" or "near X" or "by the X"

4. `getCooperationContext(chats, selfName, inventory)` — Master function that calls the three parsers above and formats a cooperation context string for planner injection. Format:
   ```
   == COOPERATION ==
   Others are doing: Jeffrey is mining, John is building
   Someone needs: Jeffrey needs iron — you have 12 iron_ingot, consider dropping some
   Build project: Jeffrey started a house near the beach — consider helping
   Suggest: Announce what YOU are doing via chat so others can coordinate
   ```
   - `inventory` is an array of `{ item: string, count: number }` — used to check if agent HAS the item someone needs
   - If no cooperation signals found, return empty string (don't inject noise)
   - Keep output under 200 chars to avoid prompt bloat

No imports except `getRecentChats` from `./chat-history.js`. Pure parsing logic. No file I/O, no LLM calls.
