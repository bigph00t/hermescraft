**Change 2: Proximity filter on chat event**

Add a proximity check function near the top of the file (after the module-level guard state variables, around line 44):
```javascript
// Phase 24: Proximity chat filter — only respond to agents within 32 blocks
const CHAT_PROXIMITY_BLOCKS = 32
function isSenderNearby(bot, senderName) {
  const selfPos = bot.entity?.position
  if (!selfPos) return true  // can't determine own position — process anyway (failsafe)
  const senderEntity = bot.players[senderName]?.entity
  if (!senderEntity?.position) return true  // sender position unknown — process anyway (startup failsafe)
  return senderEntity.position.distanceTo(selfPos) <= CHAT_PROXIMITY_BLOCKS
}
```

NOTE: The failsafe returns `true` when position is unknown — this prevents blocking all chat at startup before entities load. Per RESEARCH.md Pitfall 4: "Only block when we positively know the distance exceeds 32 blocks."

Wait -- the research says `return false` for unknown sender position. But Pitfall 4 says the failsafe should be `return true`. The CONTEXT.md doesn't specify. Use `return true` for BOTH unknown cases (own position unknown AND sender position unknown) per Pitfall 4 guidance — this is the safer behavior. We only reject when we positively know distance > 32.
