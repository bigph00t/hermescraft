Insert the proximity filter between the self-message filter and trackPlayer:
```javascript
    // Phase 24: Proximity chat — only process messages from nearby players (32 blocks)
    // Default to in-range when position unknown (human players, freshly-spawned agents)
    const senderEntity = Object.values(bot.entities || {}).find(
      e => e.type === 'player' && e.username === username
    )
    const selfPos = bot.entity?.position
    let inRange = true  // safe fallback — humans always heard
    if (senderEntity?.position && selfPos) {
      inRange = senderEntity.position.distanceTo(selfPos) <= CHAT_RANGE
    }
    if (!inRange) {
      console.log('[mind] chat from', username, 'out of range — ignored')
      return  // skip trackPlayer and respondToChat for out-of-range messages
    }
```

Key nuances per research:
- `bot.entities` is keyed by entity ID (integer), searched by username — must use Object.values().find()
- Human players may not have entity position — default `inRange = true` ensures they're always heard
- The filter goes BEFORE both trackPlayer and respondToChat — out-of-range agents are fully ignored
- console.log the skip for debugging visibility
