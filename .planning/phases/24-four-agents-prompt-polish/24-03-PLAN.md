Create a new launch-agents.sh at the project root. This is a REWRITE, not an edit of the existing launch-agents.sh (the existing one is v1 architecture targeting agent/index.js with Xvfb — completely different).

The new script must:

1. **Header and usage:**
```bash
#!/usr/bin/env bash
# launch-agents.sh — Launch N Mineflayer agents in tmux (v2 architecture)
# Usage: ./launch-agents.sh [num_agents] [server_host] [server_port]
# Default: 8 agents, localhost, 25565
set -euo pipefail
```

2. **Agent name array (all 8):**
```bash
AGENT_NAMES=(luna max ivy rust ember flint sage wren)
```

3. **Arguments:**
- $1 = number of agents to launch (default: 8, max: length of AGENT_NAMES)
- $2 = MC server host (default: localhost)
- $3 = MC server port (default: 25565)

4. **Env loading:** Same pattern as launch-duo.sh — try Glass path first, then local .env override.

5. **Env var defaults:** Same as launch-duo.sh:
```bash
VLLM_URL="${VLLM_URL:-http://localhost:8000/v1}"
VLLM_API_KEY="${VLLM_API_KEY:-not-needed}"
MODEL_NAME="${MODEL_NAME:-hermes}"
TICK_MS="${TICK_MS:-3000}"
TEMPERATURE="${TEMPERATURE:-0.6}"
MAX_TOKENS="${MAX_TOKENS:-128}"
```
CRITICAL: MAX_TOKENS defaults to 128, per locked decision — DO NOT change this.

6. **tmux session:** Name it `hermescraft-agents` (not `hermescraft-duo`).

7. **Data-driven loop:** For each agent (up to NUM_AGENTS), create a tmux window and launch with:
- AGENT_NAME and MC_USERNAME set to the agent name
- AGENT_MODE=open_ended
- All other env vars from the loaded config
- The same auto-restart loop pattern from launch-duo.sh (exit 0 = stop, exit 42 = scheduled restart, else crash restart in 5s)
- Entry point: `node $SCRIPT_DIR/start.js` (v2 Mineflayer, NOT agent/index.js)

8. **Stagger:** 30 seconds sleep between launching each agent (locked decision). First agent has no stagger (or just 3s like launch-duo.sh). Log which agent is starting and the countdown.

9. **Placeholder replacement:** Use the same HEREDOC + sed placeholder pattern from launch-duo.sh to inject env vars into the tmux command. The loop iterates AGENT_NAMES and substitutes AGENT_NAME_PLACEHOLDER, MC_HOST_PLACEHOLDER, etc.

10. **Completion banner:** List all launched agents with their tmux window names.

The script must use the launch-duo.sh pattern for env handling and the restart loop but generalize it to N agents with a data-driven loop. Do NOT use Xvfb, do NOT reference agent/index.js, do NOT reference HermesBridge ports.

Make the file executable (chmod +x).
