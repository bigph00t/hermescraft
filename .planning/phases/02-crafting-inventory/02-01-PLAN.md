  <action>
    Create body/skills/craft.js:

    1. Header: `// craft.js — Resolve full crafting dependency chain and execute each step`
    2. Imports:
       ```javascript
       import minecraftData from 'minecraft-data'
       import { solveCraft } from '../crafter.js'
       import { normalizeItemName } from '../normalizer.js'
       import { navigateToBlock } from '../navigate.js'
       import { placeBlock, FACE } from '../place.js'
       import { isInterrupted } from '../interrupt.js'
       const mcData = minecraftData('1.21.1')
       ```
    3. Internal helper `getInventorySnapshot(bot)`:
       - Iterate `bot.inventory.items()`, accumulate `{ [item.name]: count }` map
       - Return the map
    4. Internal helper `findOrPlaceCraftingTable(bot)`:
       - Search for existing crafting_table block: `bot.findBlock({ matching: mcData.blocksByName['crafting_table'].id, maxDistance: 32 })`
       - If found, navigate to it with `navigateToBlock(bot, tableBlock)`, return tableBlock
       - If not found, check inventory for crafting_table item: `bot.inventory.findInventoryItem(mcData.itemsByName['crafting_table'].id)`
       - If no item either, return null
       - Place crafting table: try 4 cardinal offsets `[offset(1,-1,0), offset(-1,-1,0), offset(0,-1,1), offset(0,-1,-1)]` — for each, get reference block via `bot.blockAt(bot.entity.position.offset(...))`, attempt `placeBlock(bot, ref, FACE.TOP)`, break on first success
       - After placing, re-find the crafting_table block (it should be adjacent now)
       - Navigate to it, return the block
       - Check `isInterrupted(bot)` after every await
    5. Main export `async function craft(bot, itemName, count = 1)`:
       - Normalize itemName via `normalizeItemName(itemName)`
       - Build inventory snapshot: `getInventorySnapshot(bot)`
       - Call `solveCraft(normalizedName, inventory)` — get `{ steps, missing }`
       - If `missing.length > 0`, return `{ success: false, reason: 'missing_materials', missing, item: normalizedName }`
       - If `steps.length === 0`, return `{ success: true, reason: 'already_have', item: normalizedName }`
       - For each step in steps:
         - Check `isInterrupted(bot)` — return interrupted if set
         - Look up itemEntry via `mcData.itemsByName[step.item]` — return unknown_item if not found
         - If `step.needsTable`:
           - Call `findOrPlaceCraftingTable(bot)`
           - If null, try to craft a crafting_table first (4 oak_planks -> crafting_table): check if bot has 4+ oak_planks, if so `bot.recipesFor(mcData.itemsByName['crafting_table'].id, null, 1, null)` and `bot.craft(recipe, 1, null)`, then retry findOrPlaceCraftingTable
           - If still null, return `{ success: false, reason: 'no_crafting_table', item: step.item }`
           - Set `tableBlock` to the found/placed block
         - else `tableBlock = null`
         - Bridge to prismarine-recipe: `const recipes = bot.recipesFor(itemEntry.id, null, 1, tableBlock)` (use 1 for minResultCount check, not step.count — step.count is how many times to repeat)
         - If `recipes.length === 0`, return `{ success: false, reason: 'no_craftable_recipe', item: step.item }`
         - Execute: `await bot.craft(recipes[0], step.count, tableBlock)`
         - Check interrupt after craft
         - Log: `console.log('[craft] crafted ${step.count}x ${step.item}')`
       - Return `{ success: true, item: normalizedName, count }`
    6. Return format matches established pattern: `{ success: boolean, reason?: string, item: string, ... }`
