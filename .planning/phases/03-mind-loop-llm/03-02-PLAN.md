  <action>
Create `mind/index.js` — the entry point that registers three event sources and routes them into a shared `think()` function.

**Module-level state:**
```javascript
let lastActionTime = Date.now()
let skillRunning = false
let thinkingInFlight = false
let idleCheckTimer = null
```

**think(bot, context) — the core function (NOT exported):**
1. Guard: `if (thinkingInFlight) return` — prevents concurrent LLM queries (Pitfall 6)
2. Set `thinkingInFlight = true`
3. In a try block:
   a. Build system prompt: `const systemPrompt = buildSystemPrompt(bot)`
   b. Build user message: `const userMessage = buildUserMessage(bot, context.trigger, context)`
   c. Log: `console.log('[mind] thinking...', context.trigger)`
   d. Query LLM: `const result = await queryLLM(systemPrompt, userMessage)`
   e. Log reasoning: `if (result.reasoning) console.log('[mind] reasoning:', result.reasoning.slice(0, 200))`
   f. If `result.command` is null: log `'[mind] no command in response, treating as idle'`, set `lastActionTime = Date.now()`, return
   g. If `result.command === 'idle'`: log, set `lastActionTime = Date.now()`, return
   h. Otherwise dispatch the command:
      - Log: `console.log('[mind] dispatching:', result.command, result.args)`
      - Set `skillRunning = true`
      - `const skillResult = await dispatch(bot, result.command, result.args)`
      - Set `skillRunning = false`
      - Log result: `console.log('[mind] skill result:', result.command, skillResult.success ? 'OK' : skillResult.reason)`
      - Set `lastActionTime = Date.now()`
      - Fire skill_complete trigger (recursive think call with skill_complete context):
        ```javascript
        // Don't await — let it fire asynchronously so the idle timer can catch long gaps
        think(bot, { trigger: 'skill_complete', skillName: result.command, skillResult })
        ```
        Wait — this would be recursive. Instead, use `setImmediate` or `setTimeout(0)` to schedule the next think:
        ```javascript
        setTimeout(() => think(bot, { trigger: 'skill_complete', skillName: result.command, skillResult }), 0)
        ```
4. In the finally block:
   - `thinkingInFlight = false`
   - If an error in the dispatch section left skillRunning true: `skillRunning = false`

**Error handling inside think():**
- Wrap the entire body in try/catch/finally
- On error: `console.error('[mind] think error:', err.message)`
- `finally`: always set `thinkingInFlight = false` and `skillRunning = false` (if it was set by this call)

**initMind(bot) — the exported init function:**

1. **Trigger 1: Chat received**
   ```javascript
   bot.on('messagestr', (msgStr, position, jsonMsg, sender) => {
     // Filter own messages and system messages (no sender)
     if (!sender || sender === bot.username) return
     // Resolve sender UUID to username
     const username = bot.players?.[sender]?.username || sender.toString().slice(0, 8)
     // Filter bot's own messages that come back with a sender UUID
     if (username === bot.username) return
     console.log('[mind] chat from', username, ':', msgStr)
     think(bot, { trigger: 'chat', sender: username, message: msgStr })
   })
   ```

2. **Trigger 2: Skill complete** — handled via setTimeout(0) inside think() after dispatch returns (see above). No separate event listener needed.

3. **Trigger 3: Idle timeout**
   ```javascript
   idleCheckTimer = setInterval(() => {
     if (skillRunning || thinkingInFlight) return
     const idleMs = Date.now() - lastActionTime
     if (idleMs >= 2000) {
       think(bot, { trigger: 'idle', idleMs })
     }
   }, 500)
   ```
   The 500ms interval just polls the idle counter. It does NOT artificially cap LLM call rate.

4. **Death handler** — clear conversation and reset state:
   ```javascript
   bot.on('death', () => {
     console.log('[mind] bot died — clearing conversation')
     clearConversation()
     lastActionTime = Date.now()  // Reset idle timer
   })
   ```

5. Log init complete: `console.log('[mind] initialized — listening for chat, skill complete, idle')`

6. Return nothing (void).

Export: `export async function initMind(bot)`

**Key design constraints (from CONTEXT.md and STATE.md):**
- NO ARTIFICIAL DELAYS: The Mind thinks as fast and often as possible. No cooldowns between think() calls.
- NO ARTIFICIAL CAPS: No hardcoded turn limits on how often think() can fire.
- Event-driven: think() fires on events (chat, skill complete, idle), NOT on a fixed tick.
- The 500ms setInterval is ONLY a sentinel for idle detection, NOT a think rate limiter.
- Chat trigger is higher priority than idle — if chat comes in during a skill, it should trigger think() which will interrupt the skill via dispatch().

File follows project conventions: ES Modules, 2-space indent, no-semi, single quotes, named exports only, one-line module header comment `// index.js -- Event-driven Mind loop: chat/skill/idle -> think() -> dispatch()`.
