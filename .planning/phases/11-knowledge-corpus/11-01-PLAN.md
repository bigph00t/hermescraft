**SMELT_FROM table** — hardcoded map of smelting products to their input (minecraft-data has NO furnace recipe data):
```javascript

  iron_ingot: 'raw_iron',
```

**flattenRecipe(recipe)** — extract unique ingredient item IDs from either shaped (`inShape`) or shapeless (`ingredients`) recipes. Filter out null slots. Return array of numeric IDs.

**resolveIngredients(itemId, visited)** — recursive DFS:
1. Get item name from `mcData.items[itemId]`
2. If `visited` has this name, return `"{name} (cycle)"`
3. Add name to visited
4. If name is in `SMELT_FROM`, return `"{name} (smelt {input})"` — this short-circuits metals to their ore
5. Look up `mcData.recipes[itemId]` — if no recipe, return `"{name} (gather/mine)"`
6. Pick simplest recipe: `recipes.reduce()` comparing `flattenRecipe(r).length`, pick fewest ingredients. If tied on count, prefer recipe whose first ingredient resolves to an item containing "oak" (for wood variant normalization)
7. Get unique ingredient names, recursively resolve each with `new Set(visited)` (copy, NOT shared reference — prevents sibling blocking)
8. Return formatted string: `"{name} (craft from: {ingredients}) -- {sub-chains}"`

**buildRecipeChunks()** — iterate `Object.keys(mcData.recipes)`, resolve each, create chunk:
```javascript
{
  id: `recipe_${item.name}`,
  text: `Recipe: ${item.displayName}\n${chainText}`,
  type: 'recipe',
  tags: ['recipe', item.name],
  source: 'minecraft-data',
}
```

Cap chain text to ~150 tokens (~600 chars). If text exceeds 600 chars, truncate sub-chains to first level only.
