  <action>
Add two new exported functions to `agent/locations.js`:

1. `getUnexploredDirection(currentPos)` — Determines which cardinal direction the agent has explored least. Logic:
   - Look at all saved locations (excluding danger zones)
   - For each location, compute which quadrant it's in relative to home (or currentPos if no home): north (+z), south (-z), east (+x), west (-x)
   - Count locations per quadrant
   - Return the quadrant with fewest locations as `{ direction: 'north'|'south'|'east'|'west', count: number }`
   - If no locations at all, return `{ direction: 'north', count: 0 }` (default)
   - Tie-break: pick randomly among tied directions using `Math.random()`

2. `getExplorationStats(currentPos)` — Returns a summary string for the planner. Format:
   ```
   Explored: 5 known locations. Least explored: west (0 locations). Farthest from home: 142 blocks (cave-1).
   ```
   - Count non-danger, non-home locations
   - Include unexplored direction
   - Find farthest named location from home (if home exists)
   - If fewer than 2 non-home locations, add "You haven't explored much yet. Consider venturing out in a new direction."

Both functions are pure -- they only read the existing `locations` object (module-level variable). No file I/O.

Also add a helper `parseLocationFromChat(message)` (exported) that extracts coordinates from chat messages like "found iron at 200,40,100" or "there's a cave at x=200 y=40 z=100". Returns `{ x, y, z }` or `null`. Regex patterns to try:
   - `at\s+(-?\d+)\s*,\s*(-?\d+)\s*,\s*(-?\d+)`
   - `at\s+x\s*=?\s*(-?\d+)\s+y\s*=?\s*(-?\d+)\s+z\s*=?\s*(-?\d+)`
   - `\((-?\d+)\s*,\s*(-?\d+)\s*,\s*(-?\d+)\)`
