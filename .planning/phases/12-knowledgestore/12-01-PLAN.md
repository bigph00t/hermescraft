**`initKnowledgeStore(chunks)` — async, called once at startup:**

1. Set `env.cacheDir = join(__dirname, '..', '.cache', 'models')` so ONNX model caches to `.cache/models/`, not `node_modules/`.

2. Load embedding model eagerly:
   ```javascript
   embedder = await pipeline('feature-extraction', 'Xenova/all-MiniLM-L6-v2')
   ```

3. Build `chunkMap`:
   ```javascript
   chunkMap = new Map(chunks.map(c => [c.id, c]))
   ```

4. Build MiniSearch BM25 index:
   ```javascript
   keywordIndex = new MiniSearch({
     fields: ['text', 'id'],
     storeFields: ['id'],
     idField: 'id',
   })
   keywordIndex.addAll(chunks)
   ```

5. Build Vectra vector index — delete old index if exists, then create fresh (rebuild-every-startup per locked decision):
   ```javascript
   const indexPath = join(__dirname, '..', '.cache', 'knowledge-index')
   if (existsSync(indexPath)) rmSync(indexPath, { recursive: true })
   vectorIndex = new LocalIndex(indexPath)
   await vectorIndex.createIndex()
   ```

6. Embed chunks in batches of 32 and insert into Vectra:
   ```javascript
   const BATCH_SIZE = 32
   for (let i = 0; i < chunks.length; i += BATCH_SIZE) {
     const batch = chunks.slice(i, i + BATCH_SIZE)
     const texts = batch.map(c => c.text)
     const output = await embedder(texts, { pooling: 'mean', normalize: true })
     const vectors = output.tolist()
     for (let j = 0; j < batch.length; j++) {
       await vectorIndex.insertItem({
         vector: vectors[j],
         metadata: { id: batch[j].id },
       })
     }
   }
   ```

7. Log completion: `console.log('[knowledgeStore] indexed ${chunks.length} chunks (BM25 + vector)')`

**`rrf(rankedIdLists, k = 60)` — internal helper, NOT exported:**

```javascript
function rrf(rankedIdLists, k = 60) {
  const scores = new Map()
  for (const list of rankedIdLists) {
    list.forEach((id, rankIdx) => {
      const prev = scores.get(id) || 0
      scores.set(id, prev + 1 / (k + rankIdx + 1))
    })
  }
  return scores
}
```

**`retrieveKnowledge(query, topK = 8)` — async, exported:**

1. BM25 search with id field boosted 3x, fuzzy 0.2, prefix true:
   ```javascript
   const bm25Results = keywordIndex.search(query, {
     boost: { id: 3 },
     fuzzy: 0.2,
     prefix: true,
   })
   const bm25Ids = bm25Results.slice(0, 50).map(r => r.id)
   ```

2. Vector search — embed query as single string, query Vectra top 50:
   ```javascript
   const qOut = await embedder(query, { pooling: 'mean', normalize: true })
   const qVec = Array.from(qOut.data)
   const vecResults = await vectorIndex.queryItems(qVec, 50)
   const vecIds = vecResults.map(r => r.item.metadata.id)
   ```

3. RRF fusion:
   ```javascript
   const rrfScores = rrf([bm25Ids, vecIds])
   ```

4. Sort by score descending, take top-K, hydrate from chunkMap, filter undefined:
   ```javascript
   const sorted = [...rrfScores.entries()]
     .sort((a, b) => b[1] - a[1])
     .slice(0, topK)
   return sorted
     .map(([id, score]) => ({ chunk: chunkMap.get(id), score }))
     .filter(r => r.chunk !== undefined)
   ```

**IMPORTANT implementation notes from research:**
- Batch embedding (init): use `output.tolist()` which returns `number[][]`
- Single embedding (query): use `Array.from(qOut.data)` which returns `number[]` from Float32Array
- Vectra `queryItems` second arg: pass just the topK number (50), NOT an empty string — check the actual API signature in `node_modules/vectra/dist/LocalIndex.d.ts` after install and adjust if needed
- MiniSearch returns `{ id, score, match }` — only `id` is needed for RRF, chunk lookup is via `chunkMap`
- Do NOT use `deleteIfExists` option on `createIndex` — instead manually `rmSync` the folder before creating (safer, avoids API uncertainty from research open question #3)
