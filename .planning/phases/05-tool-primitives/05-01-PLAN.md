<objective>
Create the item name normalization layer and remove the mine action from the agent tool set.

Purpose: LLM-generated item names (sticks, oak_planks_4, wooden_planks) cause silent failures when they reach the mod. A normalization layer transparently corrects these before any dispatch. The mine action routes agents underground via Baritone — removing it forces surface-only block breaking via look_at_block + break_block.

Output: New normalizer.js module, updated actions.js with normalization call, updated tools.js without mine, updated prompt.js gameplay instructions.
</objective>

4. Export `normalizeItemName(name)` function with this pipeline:
   a. Return empty string if name is falsy
   b. Convert to lowercase, trim whitespace
   c. Strip `minecraft:` prefix if present
   d. Strip trailing `_N` numeric suffix via regex `/^(.+?)_(\d+)$/` — but ONLY if the base name without suffix is a valid item in mcData.itemsByName (prevents stripping legitimate suffixes like `raw_iron`)
   e. Check ALIASES map — if match, use the alias value
   f. Try simple plural stripping: if name ends with `s` and the singular form exists in mcData.itemsByName, use singular
   g. Validate against `mcData.itemsByName[result]` — if valid, return result
   h. If not valid, try prefix search: find first item in mcData.itemsByName whose key starts with result
   i. If still not found, return the input unchanged (let it fail downstream with a clear error)

     ```javascript
     if (action.item) action.item = normalizeItemName(action.item)
     if (action.blockName) action.blockName = normalizeItemName(action.blockName)
     ```
   - In `validatePreExecution()`, normalize `action.item` at the top of the function (after the null guard) so all downstream checks use canonical names:
