<objective>
Build the Paper TTS plugin that receives PCM audio via HTTP and injects it as proximity voice via Simple Voice Chat's EntityAudioChannel API, plus configure both docker-compose files for Simple Voice Chat plugin auto-install and UDP port exposure.

Purpose: The plugin is the server-side audio injection layer — the only way to make agents "speak" in-game with 3D proximity attenuation. Without it, the Python TTS bridge has no endpoint to deliver audio to.

Output: Compilable tts-plugin/ directory with Paper plugin jar, updated docker-compose files with voice chat support.
</objective>

    **tts-plugin/src/main/java/hermescraft/tts/TtsPlugin.java:**
    - Implements JavaPlugin (Paper) and registers TtsInjector as voicechat addon
    - In onEnable():
      1. Log "HermesCraftTTS plugin enabled"
      2. Register this plugin with BukkitVoicechatService: `getServer().getServicesManager().load(BukkitVoicechatService.class).registerPlugin(new TtsInjector(this))`
      3. Wrap in try-catch — log error if Simple Voice Chat not loaded yet
    - Store plugin reference for TtsInjector to access Bukkit server

    **tts-plugin/src/main/java/hermescraft/tts/TtsInjector.java:**
    - Implements `de.maxhenkel.voicechat.api.VoicechatPlugin`
    - Fields:
      - TtsPlugin plugin (reference to Paper plugin)
      - VoicechatServerApi voicechatApi (set in initialize())
      - HttpServer httpServer (JDK built-in com.sun.net.httpserver)
    - getPluginId() returns "hermescraft-tts"
    - initialize(VoicechatApi api):
      1. Cast to VoicechatServerApi, store in field
      2. Start HttpServer on port 8765 with single-thread executor
      3. Register handler at path "/tts" for POST requests
      4. Log "[tts-plugin] HTTP server started on port 8765"
    - HTTP handler /tts:
      1. Only accept POST method (return 405 for other methods)
      2. Read request body as UTF-8 string
      3. Parse JSON manually with Gson (available in Paper runtime):
         - "agent": string (agent name, e.g., "luna")
         - "pcm_b64": string (base64-encoded 48kHz 16-bit mono PCM)
      4. Look up player: `plugin.getServer().getPlayer(agentName)` — case-insensitive
      5. If player is null: return HTTP 404 with JSON {"error": "player not found"}
      6. Decode base64 PCM to byte[]
      7. Convert byte[] to short[] (little-endian): use ByteBuffer.wrap(bytes).order(LITTLE_ENDIAN).asShortBuffer().get(shorts)
      8. Create EntityAudioChannel:
         ```java
         EntityAudioChannel channel = voicechatApi.createEntityAudioChannel(
             UUID.randomUUID(),
             voicechatApi.fromEntity(player)
         );
         channel.setDistance(48);  // 48-block proximity radius
         ```
      9. Create AudioPlayer with frame supplier:
         - Frame size = 960 shorts (20ms at 48kHz) — HARDCODED constant
         - Supplier returns 960-sample frames sequentially from allSamples
         - Pad last frame to 960 with zeros if needed
         - Return null when all frames consumed (signals stop)
      10. Call audioPlayer.startPlaying()
      11. Return HTTP 200 with JSON {"ok": true, "frames": frameCount}
    - Helper method pcmBytesToShorts(byte[] bytes) -> short[]:
      ```java
      short[] shorts = new short[bytes.length / 2];
      java.nio.ByteBuffer.wrap(bytes).order(java.nio.ByteOrder.LITTLE_ENDIAN).asShortBuffer().get(shorts);
      return shorts;
      ```
    - Error handling: wrap entire handler in try-catch, return HTTP 500 with error JSON on failure
    - Use Gson for JSON parsing and response serialization (available at runtime in Paper)

    IMPORTANT NOTES:
    - Do NOT use fabric-loom — this is a Paper plugin, not a Fabric mod
    - Frame size MUST be exactly 960 shorts (48kHz * 0.020s = 960) — non-negotiable
    - Port 8765 chosen to avoid conflict with HermesBridge (3001) and LLM (8000)
    - The HttpServer runs on a separate thread; audio playback runs on Simple Voice Chat's internal thread via AudioPlayer
