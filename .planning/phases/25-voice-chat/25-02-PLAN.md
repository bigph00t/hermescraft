    **infra/tts-bridge.py:**
    Create the TTS bridge Python script with the following architecture:

    1. **Imports:** subprocess, re, threading, queue, requests, json, os, sys, base64, time, numpy, scipy.signal.resample_poly, piper.PiperVoice

    2. **Constants:**
       - AGENT_NAMES = frozenset({"luna", "max", "ivy", "rust", "ember", "flint", "sage", "wren"})
       - PLUGIN_URL = os.environ.get("TTS_PLUGIN_URL", "http://localhost:8765/tts")
       - VOICES_CONFIG = os.environ.get("TTS_VOICES_CONFIG") or default to os.path.join(os.path.dirname(__file__), "tts-voices.json")
       - MC_CONTAINER = os.environ.get("MC_CONTAINER", "hermescraft-server")
       - CHAT_RE = re.compile(r'\[.*?INFO\].*?<(\w+)> (.+)')

    3. **Voice loading (at startup, not per-request):**
       - Read tts-voices.json
       - For each agent/model_path pair: load PiperVoice.load(model_path)
       - Store in dict: voices = {agent: PiperVoice}
       - Log each loaded voice: "[tts] Loaded voice for {agent}: {model_path}"
       - If a model file is missing: log warning and skip that agent (don't crash the bridge)

    4. **synthesize_48k(agent: str, text: str) -> bytes function:**
       - Get voice from voices dict
       - Concatenate all audio chunks: `b"".join(c.audio_int16_bytes for c in voice.synthesize(text))`
       - Convert to numpy float32 array: `np.frombuffer(raw, dtype=np.int16).astype(np.float32)`
       - Resample: `resample_poly(samples_22k, up=160, down=147)` — 48000/22050 = 160/147 reduced fraction
       - Convert back: `.astype(np.int16).tobytes()`
       - Return bytes

    5. **Worker thread (synthesis + HTTP POST):**
       - job_queue = queue.Queue(maxsize=32) — bounded to prevent memory buildup
       - Worker function runs in daemon thread:
         a. Dequeue (agent, text) tuple
         b. Call synthesize_48k(agent, text)
         c. Base64 encode the PCM bytes
         d. POST to PLUGIN_URL with JSON: {"agent": agent, "pcm_b64": encoded}
         e. Log result: "[tts] {agent}: '{text[:50]}...' -> {len(pcm)} bytes, {response.status_code}"
         f. On requests.RequestException: log error, continue (don't crash)
         g. On any other exception: log error, continue
         h. Call job_queue.task_done()
       - Start worker thread as daemon

    6. **Log tail (main thread):**
       - Start subprocess: `docker logs -f {MC_CONTAINER}` with stdout=PIPE, stderr=STDOUT, text=True
       - For each line from proc.stdout:
         a. Match against CHAT_RE
         b. If match: extract sender (lowercased) and message
         c. If sender in AGENT_NAMES and sender in voices dict:
            - Try to put (sender, message) in job_queue with timeout=0.1
            - If queue is full: log warning "[tts] Queue full, dropping message from {sender}"
       - If subprocess exits: log error and retry after 5 seconds (MC server may not be ready)

    7. **Main block:**
       - Print banner: "[tts] HermesCraft TTS Bridge"
       - Print loaded voice count
       - If no voices loaded: print error and exit(1)
       - Start worker thread
       - Enter log tail loop with retry
       - Handle KeyboardInterrupt gracefully (exit 0)

    IMPORTANT CONSTRAINTS:
    - Load ALL voice models at startup — PiperVoice.load() takes 200-500ms per model. Never load per-request.
    - Resample with up=160, down=147 (NOT up=48000, down=22050 — that allocates a huge polyphase filter)
    - The worker thread must be daemon=True so the process exits cleanly on SIGTERM
    - Filter ONLY messages from AGENT_NAMES set — never synthesize human player messages or server announcements
    - Use bounded queue (maxsize=32) to prevent memory buildup if synthesis can't keep up
    - The bridge should not crash if the plugin endpoint is unreachable — log and continue
