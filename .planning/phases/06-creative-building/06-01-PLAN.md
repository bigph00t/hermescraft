Create `body/skills/build.js` following the established skill pattern from gather.js (cooperative interrupt loop, { success, reason } return struct).

**Module header:**

```
// build.js — Build structures from blueprint JSON plans with verified placement
```

**Imports:**

```javascript
import { readFileSync, writeFileSync, existsSync, mkdirSync, readdirSync } from 'fs'
import { join, dirname } from 'path'
import { fileURLToPath } from 'url'
```

**Module state:**
```javascript
const mcData = minecraftData('1.21.1')
const __dirname = dirname(fileURLToPath(import.meta.url))
const BLUEPRINTS_DIR = join(__dirname, '..', 'blueprints')

let _stateFile = ''
let _activeBuild = null
```

**Exports to implement:**

1. `initBuild(config)` — Set `_stateFile = join(config.dataDir, 'build_state.json')`. If file exists, load into `_activeBuild` with try/catch fallback to null. Matches init<Subsystem> convention.

2. `listBlueprints()` — Read BLUEPRINTS_DIR, filter *.json, parse each, return array of `{ name, description, size }` objects. Silent catch returns [].

3. `getActiveBuild()` — Returns `_activeBuild` or null. Used by prompt.js for injecting build context.

4. `getBuildProgress()` — Returns human-readable string for prompt injection. If no active build returns ''. If paused: "BUILD PAUSED: {name} ({placed}/{total}). Need: {missing}". If active: "BUILDING: {name} ({placed}/{total}, {percent}%)".

5. `build(bot, blueprintName, originX, originY, originZ)` — The main skill function:

   a. **Load blueprint:** Read `join(BLUEPRINTS_DIR, blueprintName + '.json')`, parse JSON. Return `{ success: false, reason: 'unknown_blueprint' }` if not found.

   b. **Resolve palette:** For each palette key, find first preferred block where `mcData.itemsByName[name]` exists AND `bot.inventory.findInventoryItem(mcData.itemsByName[name].id, null)` is not null. Fallback to first preferred entry, then 'cobblestone'. Store as `resolvedPalette` map (char -> blockName string).

   c. **Build block queue:** Sort layers by y ascending. For each layer, iterate rows (row index = z offset), then columns (col index = x offset). Skip chars that are '.' or ' '. Look up char in resolvedPalette. Push `{ x: originX + col, y: originY + layer.y, z: originZ + row, block: resolvedPalette[char] }` into queue array.

   d. **Pre-flight area check:** Scan ground level at originY. Count non-air blocks in the footprint (originX to originX + size.x - 1, originZ to originZ + size.z - 1). If more than 50% are non-air, return `{ success: false, reason: 'site_blocked', message: 'Too many blocks at build site. Choose flatter terrain.' }`.

   e. **Resume support:** If `_activeBuild` exists and matches blueprintName + origin, use its `completedIndex` to skip already-placed blocks. Otherwise start fresh. At resume, also filter queue entries where `bot.blockAt(new Vec3(entry.x, entry.y, entry.z))?.name !== 'air'` — those are already placed.

   f. **Save initial state:**
   ```javascript
   _activeBuild = {
     blueprintName,
     origin: { x: originX, y: originY, z: originZ },
     totalBlocks: queue.length,
     completedIndex: 0,
     paused: false,
     missingMaterials: [],
     startedAt: Date.now(),
   }
   _saveState()
   ```

   g. **Placement loop:** Starting from `_activeBuild.completedIndex`, iterate through queue:
   - Check `isInterrupted(bot)` — if true, `_saveState()` and return `{ success: false, reason: 'interrupted', placed: _activeBuild.completedIndex, total: _activeBuild.totalBlocks }`
   - Get block entry from queue at current index
   - Check inventory: `const itemMeta = mcData.itemsByName[entry.block]` then `bot.inventory.findInventoryItem(itemMeta?.id, null)`. If not found, compute all missing materials from remaining queue, set `_activeBuild.paused = true`, `_activeBuild.missingMaterials = [...missingSet]`, `_saveState()`, return `{ success: false, reason: 'missing_material', missing: [...missingSet], placed: _activeBuild.completedIndex, total: _activeBuild.totalBlocks }`
   - **Navigate:** `await navigateTo(bot, entry.x, entry.y, entry.z, 3, 15000)` — 3 block range, 15s timeout. If fails, skip this block (increment completedIndex, continue) — terrain might make it unreachable.
   - Check `isInterrupted(bot)` again after nav
   - **Equip:** `await bot.equip(itemMeta.id, 'hand')` wrapped in try/catch (non-fatal)
   - Check `isInterrupted(bot)` after equip
   - **Find reference block:** Try these positions in order for an adjacent solid block:
     1. `(x, y-1, z)` — below (most common: floor or block below)
     2. `(x-1, y, z)` — west
     3. `(x+1, y, z)` — east
     4. `(x, y, z-1)` — north
     5. `(x, y, z+1)` — south
     For each, call `bot.blockAt(new Vec3(rx, ry, rz))`. Use the first one that is not null and not air. Compute the face vector as `new Vec3(entry.x - ref.position.x, entry.y - ref.position.y, entry.z - ref.position.z)`.
     If no solid reference found, skip this block (log warning, increment completedIndex, continue).
   - **Place:** Call `placeBlock(bot, refBlock, faceVector)`. If success, increment `_activeBuild.completedIndex`, `_saveState()` every 5 blocks (not every block — reduces disk writes). If place fails, log and skip (increment completedIndex, continue).
   - Check `isInterrupted(bot)` after place

   h. **Completion:** After loop finishes, set `_activeBuild = null`, delete state file if exists. Return `{ success: true, placed: totalPlaced, total: queue.length, blueprintName }`.

6. `_saveState()` (not exported) — `writeFileSync(_stateFile, JSON.stringify(_activeBuild, null, 2))` wrapped in try/catch.
