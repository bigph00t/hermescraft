Add two new exports to agent/locations.js without modifying any existing functions:

**recordDeathLocation(x, y, z, cause, lesson):**
- Stores death location in the existing `locations` object with a special naming convention: key is `danger-{N}` where N is the count of existing danger entries + 1
- Value shape: `{ x: Math.round(x), y: Math.round(y), z: Math.round(z), type: 'danger', cause, lesson, saved: new Date().toISOString() }`
- Cap at 10 danger zones maximum. If at 10, remove the oldest one (lowest N) before adding new one. Implement by finding all keys starting with "danger-", sorting by saved timestamp, and removing the oldest.
- Call `saveLocations()` immediately after adding (deaths are critical data)

**getNearbyDangers(position, radius = 30):**
- Takes agent's current position `{ x, y, z }` and a radius (default 30 blocks)
- Scans all location entries where `type === 'danger'`
- Returns an array of `{ name, x, y, z, cause, lesson, distance }` for all danger zones within radius
- Distance calculated as 3D Euclidean: `Math.sqrt(dx*dx + dy*dy + dz*dz)`
- Returns empty array if no danger zones within radius or if position is null

Do NOT modify existing functions. Do NOT change the locations object structure. The danger entries are just regular location entries with type='danger' and extra fields (cause, lesson).
