**mind/social.js** â€” Port agent/social.js with these changes:

1. `initSocial(config)` â€” loads `players.json` from `config.dataDir`. Creates dir if needed. If `config.partnerName` is set, seeds a partner entry in players if not already present:
   ```javascript
   if (config.partnerName && !players[config.partnerName]) {
     players[config.partnerName] = {
       first_met: new Date().toISOString(),
       interactions: [],
       notable_interactions: [],
       sentiment: 3,       // start as acquaintance, not stranger
       relationship: 'acquaintance',
     }
   }
   ```
2. `trackPlayer(name, interaction)` â€” same logic as v1. Interaction types: 'chat', 'gave_item', 'attacked', 'near'. Sentiment scoring: chat +0.5, gave_item +2, attacked -3. Decay based on hours since last seen. Keep last 20 interactions. Relationship thresholds: friend >= 5, acquaintance >= 2, hostile <= -3, else stranger.
3. `getPlayersForPrompt(bot)` â€” CHANGED signature from v1 (v1 takes nearbyEntities array). This version takes the bot object directly and reads `bot.players` + `bot.entities` to determine who is nearby. Returns "Known players:\n..." text with relationship, sentiment, nearby status, last chat. Only show non-strangers or nearby players.
4. `savePlayers()` â€” writes players.json.
5. `getPartnerLastChat(partnerName)` â€” NEW helper. Returns `{ sender, message, timestamp }` or null for the partner's most recent chat interaction. Used by prompt builder to inject partner context.
6. File header: `// social.js -- Player relationship tracking, sentiment scoring, partner awareness`
7. Follow conventions: ES Modules, 2-space indent, no-semi, single quotes.

5. `getLocationsForPrompt(botPosition)` â€” returns compact text listing known locations with distance and cardinal direction from bot's current position. 8. File header: `// locations.js -- Named location memory: home, waypoints, prompt injection`
9. Follow conventions: ES Modules, 2-space indent, no-semi, single quotes.
