**body/skills/harvest.js** (GPL-02):
- Module header: `// harvest.js â€” Detect and harvest mature crops, then replant`
- Import: `minecraftData`, `navigateTo` from `../navigate.js`, `isInterrupted` from `../interrupt.js`
- Define `CROP_CONFIG` object mapping crop block names to their mature age and seed item:
  - `wheat`: mature age 7, seed `wheat_seeds`
  - `carrots`: mature age 7, seed `carrot`
  - `potatoes`: mature age 7, seed `potato`
  - `beetroots`: mature age 3, seed `beetroot_seeds`
- Export async function `harvest(bot, cropName = 'wheat', count = 8)`:
  - Determine target crop from cropName (normalize: 'wheat' -> 'wheat', 'wheat_seeds' -> 'wheat', etc.)
  - Find mature crop blocks using `bot.findBlocks({ matching: block => block.name === cropBlockName && block.getProperties().age === matureAge, maxDistance: 32, count })` â€” use the lambda form to handle block state (Pitfall 3 from research)
  - If no mature crops found, return `{ success: false, harvested: 0, replanted: 0, reason: 'no_mature_crops_nearby' }`
  - For each found crop block (up to count):
    - Check `isInterrupted(bot)` â€” break if true
    - Navigate to within 3 blocks of the crop position using `navigateTo(bot, x, y, z, 3)`
    - If nav fails, skip this crop (continue loop)
    - Dig the crop block: `await bot.dig(bot.blockAt(cropPos))`
    - Replant: find the seed item in inventory (`bot.inventory.items().find(i => i.name === seedItem)`), if available, equip it and `await bot.activateBlock(bot.blockAt(cropPos.offset(0, -1, 0)))` to plant on the farmland below
    - Increment harvested counter; increment replanted counter if replant succeeded
  - Return `{ success: true, harvested, replanted }`
  - Wrap everything in try/catch, return `{ success: false, reason: err.message }` on error

**body/skills/hunt.js** (GPL-03):
- Module header: `// hunt.js â€” Seek hostile mobs within expanded radius and engage in combat`
- Import: `HOSTILE_MOBS`, `combatLoop` from `./combat.js`, `navigateTo` from `../navigate.js`, `isInterrupted` from `../interrupt.js`
- Define `HUNT_RADIUS = 48` (wider than combat's 16b reactive radius)
- Export async function `hunt(bot, targetType = null)`:
  - Find hostile mob entities within HUNT_RADIUS: iterate `Object.values(bot.entities)`, filter by `e.type === 'mob' && HOSTILE_MOBS.has(e.name) && distance < HUNT_RADIUS`
  - If targetType provided, further filter to `e.name === targetType`
  - Sort by distance (closest first)
  - If no target found, return `{ success: false, reason: 'no_hostile_nearby', searched: HUNT_RADIUS }`
  - Take the closest target
  - Log: `console.log('[hunt] seeking ${target.name} at distance ${dist.toFixed(1)}')`
  - Navigate toward target position (use navigateTo with range=4 to get in melee range)
  - Check `isInterrupted(bot)` after nav
  - If target still valid and in range, call `combatLoop(bot, target)` and return its result with `{ ...combatResult, target: target.name }`
  - If target despawned during nav, return `{ success: false, reason: 'target_despawned' }`
  - Wrap in try/catch

**body/skills/explore.js** (GPL-04):
- Module header: `// explore.js â€” Navigate to unexplored areas and scan for notable features`
- Import: `navigateTo` from `../navigate.js`, `isInterrupted` from `../interrupt.js`
- Define `EXPLORE_DISTANCE = 64` (blocks to travel per explore call)
- Define `NOTABLE_BLOCKS` Set: `['chest', 'spawner', 'bookshelf', 'enchanting_table', 'anvil', 'brewing_stand', 'beacon', 'end_portal_frame', 'nether_portal', 'bell', 'lectern', 'fletching_table', 'smithing_table', 'cartography_table', 'loom', 'barrel', 'blast_furnace', 'smoker', 'composter', 'stonecutter']` â€” these indicate villages, dungeons, or structures
- Export async function `explore(bot, direction = null, distance = 64)`:
  - Compute target position based on direction:
    - If direction is 'north': target = pos + (0, 0, -distance)
    - 'south': (0, 0, +distance), 'east': (+distance, 0, 0), 'west': (-distance, 0, 0)
    - If direction is null: pick a random cardinal direction
    - Use bot.entity.position as base; keep y = pos.y (same altitude)
  - Navigate to target: `const navResult = await navigateTo(bot, targetX, pos.y, targetZ, 8)`
  - Only proceed with discovery scan if navigation at least partially succeeded (bot moved) â€” check `bot.entity.position.distanceTo(startPos) > 10` (Pitfall 4: don't log false discoveries if bot didn't move)
  - Scan for discoveries within 16 blocks of new position:
    - Notable blocks: `bot.findBlocks({ matching: block => NOTABLE_BLOCKS.has(block.name), maxDistance: 16, count: 5 })` â€” return block name and position
    - Nearby entities: filter `bot.entities` for villagers (`e.name === 'villager'`), animals, hostile mobs within 16b
  - Build discoveries array: `[{ type: 'structure'|'entity', name, x, z, description }]`
  - Return `{ success: true, discoveries, newPos: { x: Math.round(pos.x), y: Math.round(pos.y), z: Math.round(pos.z) }, direction: dirUsed }`
  - If navigation failed completely (bot didn't move), return `{ success: false, reason: 'navigation_failed', direction: dirUsed }`
  - Wrap in try/catch
