Create three new ES module files following the exact patterns in memory.js and social.js (init with agentConfig, file paths via join(config.dataDir, ...), silent catch on I/O errors).

**agent/autobiography.js** — Autobiographical event log (D-02: autobiography.jsonl)
- `initAutobiography(agentConfig)` — set DATA_DIR, create file path `join(dataDir, 'autobiography.jsonl')`, load existing entries on startup.
- `recordEvent(event)` — append a JSONL entry. Event shape: `{ timestamp: ISO string, gameDay: number, type: string, description: string, location: {x,y,z}, importance: number }`. Event types: 'death' (importance 10), 'build_complete' (8), 'new_player' (7), 'found_resource' (5), 'chat_highlight' (4), 'routine' (1). Use appendFileSync like writeSessionEntry in memory.js. Also push to in-memory array. Cap in-memory array at 100 entries (shift oldest when full, per Claude's Discretion in CONTEXT.md).
- `getRecentEvents(n = 10)` — return last N events from in-memory array.
- `getEventsSummary()` — return a compact text summary: group by gameDay, format as "Day 1: built shelter, found iron. Day 2: met Jeffrey, died to skeleton." Max 10 days, 3 events per day. This is what the planner will read.

**agent/chests.js** — Chest inventory tracking (D-04: chests.json)
- `initChests(agentConfig)` — set file path `join(dataDir, 'chests.json')`, load existing data.
- `trackChest(x, y, z, contents)` — store/update chest contents keyed by coordinate string `"${x},${y},${z}"`. Contents is an array of `{item: string, count: number}`. Write to disk immediately (chests change infrequently, no batching needed).
- `getChestContents(x, y, z)` — return contents array for a specific chest, or null.
- `getChestsForPrompt()` — return text summary: "Known chests: (10,64,20): oak_planksx12, stickx4 | (15,64,25): iron_orex3". Return empty string if no chests tracked.
- `saveChests()` — explicit save (called by periodicSave).

**agent/chat-history.js** — Chat message history (D-03, D-10: chat-history.jsonl)
- `initChatHistory(agentConfig)` — set file path `join(dataDir, 'chat-history.jsonl')`, load existing entries (last 50 only, per D-03).
- `recordChat(sender, message)` — append JSONL entry `{ timestamp: ISO string, sender: string, message: string }`. Also push to in-memory ring buffer. Keep max 50 entries in memory and on disk. When disk file exceeds 50 lines, rewrite with only the last 50.
- `getRecentChats(n = 10)` — return last N entries from ring buffer.
- `getChatSummary()` — return a compact text summary for the planner: group by sender, show last 2-3 messages each. Format: "Jeffrey said: 'nice house' (2min ago), 'got any wood?' (5min ago)". Use relative timestamps from Date.now().

All three modules: use `import { readFileSync, writeFileSync, appendFileSync, mkdirSync, existsSync } from 'fs'` and `import { join, dirname } from 'path'`. Silent catches on all I/O (agent must never crash from memory I/O failure). No external dependencies.
