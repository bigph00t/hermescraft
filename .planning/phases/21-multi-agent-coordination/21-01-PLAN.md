Create two new modules following codebase conventions (camelCase exports, named only, module header comment, `_` prefix for module state, init pattern).

**mind/taskRegistry.js** (COO-01):
- Module header: `// taskRegistry.js — Shared task registry: claim/release tasks to prevent duplicate work across agents`
- Module state: `let _sharedDir = ''`, `let _registryFile = ''`, `let _agentName = ''`
- `initTaskRegistry(config)` — sets `_agentName = config.name`, computes `_sharedDir = join(dirname(config.dataDir), 'shared')`, creates dir with `mkdirSync(_sharedDir, { recursive: true })`, sets `_registryFile = join(_sharedDir, 'task-registry.json')`. On init, also calls `_releaseStaleOwn()` to clean up any claims this agent held from a previous crashed session (10-minute TTL).
- `_loadRegistry()` internal — try `JSON.parse(readFileSync(_registryFile, 'utf-8'))` catch return `{ tasks: [], lastUpdated: null }`. This handles ENOENT on cold start.
- `_writeRegistry(data)` internal — `data.lastUpdated = new Date().toISOString()`, write to `_registryFile + '.tmp'`, then `renameSync(tmp, _registryFile)`.
- `registerTask(taskId, description, meta = {})` — loads registry, pushes `{ id: taskId, description, claimedBy: null, claimedAt: null, status: 'pending', meta, createdAt: new Date().toISOString() }`, writes registry. Returns the task object.
- `claimTask(agentName, taskId)` — loads registry, finds task by id. If not found or `claimedBy !== null` (and claim is not stale), return false. Sets `claimedBy = agentName`, `claimedAt = new Date().toISOString()`, `status = 'active'`. Writes registry. **Then re-reads and verifies** `claimed.claimedBy === agentName` (optimistic concurrency per research Pitfall 1). Returns boolean.
- `releaseTask(agentName, taskId)` — loads registry, finds task where `id === taskId && claimedBy === agentName`, sets `claimedBy = null`, `claimedAt = null`, `status = 'pending'`. Writes registry.
- `completeTask(agentName, taskId)` — loads, finds task where `id === taskId && claimedBy === agentName`, sets `status = 'done'`. Writes.
- `listTasks(filter = {})` — loads registry, returns `data.tasks` optionally filtered by `filter.status` or `filter.claimedBy`.
- `_releaseStaleOwn()` internal — loads registry, finds tasks where `claimedBy === _agentName` and `claimedAt` is older than 10 minutes, sets those to `claimedBy = null, status = 'pending'`. Writes if any changed. Logs `[task-registry] released N stale claims from previous session`.
- Stale claim detection: in `claimTask`, if a task's `claimedBy !== null` but `claimedAt` is older than 10 minutes, treat it as expired and allow re-claiming. This handles Pitfall 5 (crash recovery).
- Imports: `readFileSync, writeFileSync, renameSync, mkdirSync` from `fs`, `join, dirname` from `path`.

**mind/coordination.js** (COO-04):
- Module header: `// coordination.js — Activity broadcasting: each agent writes its status, reads partner's`
- Module state: `let _activityFile = ''`, `let _activityTmp = ''`, `let _partnerFile = ''`, `let _agentName = ''`
- `initCoordination(config)` — sets `_agentName = config.name`, computes shared dir as `join(dirname(config.dataDir), 'shared')`, creates with `mkdirSync(sharedDir, { recursive: true })`, sets `_activityFile = join(sharedDir, 'activity-' + config.name + '.json')`, `_activityTmp = _activityFile + '.tmp'`, `_partnerFile = join(sharedDir, 'activity-' + config.partnerName + '.json')`.
- `broadcastActivity(command, args = {}, status = 'running')` — writes `{ agent: _agentName, command, args, status, ts: Date.now() }` atomically to `_activityFile` via writeFileSync(tmp) + renameSync(tmp, target).
- `getPartnerActivityForPrompt()` — try readFileSync `_partnerFile`, parse JSON. Compute `ageS = Math.round((Date.now() - data.ts) / 1000)`. If `ageS > 120`, return `null` (stale — partner offline per Pitfall 4). Return formatted string: `"<partner> is <status>: <command> <truncated args> (<ageS>s ago)"` for non-idle, or `"<partner> is idle (<ageS>s ago)"` for idle status. Catch returns `null` (ENOENT or parse error = partner not yet active).
- Imports: `readFileSync, writeFileSync, renameSync, mkdirSync` from `fs`, `join, dirname` from `path`.
