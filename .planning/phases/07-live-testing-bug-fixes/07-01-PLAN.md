**body/normalizer.js** — The ALIASES map conflates item names with block names. `'iron_ore': 'raw_iron'` and `'gold_ore': 'raw_gold'` are correct for ITEM normalization (crafting, smelting) but WRONG for BLOCK normalization (mining, gathering). In minecraft-data 1.21.1:
- `iron_ore` IS a valid block name (blocksByName has it)
- `raw_iron` is NOT a valid block name (only in itemsByName)
- Same for `gold_ore` / `raw_gold`

The fix: split the ALIASES map into ITEM_ALIASES and BLOCK_ALIASES (or a shared COMMON_ALIASES plus type-specific ones). The `_normalize` function already receives the registry as a parameter, so the fix is:

1. Create separate alias maps:
   - `COMMON_ALIASES` — aliases valid for both items and blocks (sticks, planks, wood, cobble, wooden tools, etc.)
   - `ITEM_ONLY_ALIASES` — aliases that only apply to item normalization: `'iron_ore': 'raw_iron'`, `'gold_ore': 'raw_gold'`
   - `BLOCK_ONLY_ALIASES` — currently empty but available for future use

2. Add a second parameter to `_normalize` for which alias set to use, OR determine it from the registry parameter:
   - When registry === mcData.itemsByName, use COMMON_ALIASES merged with ITEM_ONLY_ALIASES
   - When registry === mcData.blocksByName, use COMMON_ALIASES only (skip iron_ore/gold_ore remapping)

Simplest approach: keep a single ALIASES map but add a `registryType` parameter:

```javascript
const BLOCK_REGISTRY = mcData.blocksByName
const ITEM_REGISTRY = mcData.itemsByName

// Aliases that should NOT apply when normalizing block names
const ITEM_ONLY_ALIASES = new Set(['iron_ore', 'gold_ore'])

function _normalize(name, registry) {
  // ... existing steps 1-4 ...

  // Step 5: check ALIASES map — skip item-only aliases when normalizing blocks
  if (ALIASES[result]) {
    if (registry === BLOCK_REGISTRY && ITEM_ONLY_ALIASES.has(result)) {
      // iron_ore IS a valid block name — don't remap to raw_iron
      // Fall through to registry validation (step 7)
    } else {
      return ALIASES[result]
    }
  }

  // ... rest unchanged ...
}
```
