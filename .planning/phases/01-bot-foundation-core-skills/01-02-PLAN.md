  <action>
    Create `body/navigate.js` with this exact export:

    ```javascript
    // navigate.js — Wall-clock timeout wrapper around mineflayer-pathfinder goto()
    import { goals } from 'mineflayer-pathfinder'
    ```

    Export `navigateTo(bot, x, y, z, range = 1, timeoutMs = 30000)`:
    - Creates a `goals.GoalNear(x, y, z, range)` goal
    - Wraps `bot.pathfinder.goto(goal)` in `Promise.race` with a `setTimeout` reject:
      ```javascript
      const timer = new Promise((_, reject) =>
        setTimeout(() => reject(new Error('nav_timeout')), timeoutMs)
      )
      ```
    - On success: return `{ success: true }`
    - On timeout (`err.message === 'nav_timeout'`): call `bot.pathfinder.setGoal(null)` to force immediate halt, then return `{ success: false, reason: 'nav_timeout' }`
    - On pathfinder error (e.g., 'noPath'): call `bot.pathfinder.setGoal(null)`, return `{ success: false, reason: err.message }`
    - CRITICAL: Must clear the setTimeout timer on success to avoid lingering timers. Use a `clearTimeout` pattern:
      ```javascript
      let timerId
      const timer = new Promise((_, reject) => {
        timerId = setTimeout(() => reject(new Error('nav_timeout')), timeoutMs)
      })
      try {
        await Promise.race([bot.pathfinder.goto(goal), timer])
        clearTimeout(timerId)
        return { success: true }
      } catch (err) {
        clearTimeout(timerId)
        bot.pathfinder.setGoal(null)
        return { success: false, reason: err.message === 'nav_timeout' ? 'nav_timeout' : err.message }
      }
      ```

    Also export `navigateToBlock(bot, block, timeoutMs = 30000)` as a convenience:
    - Calls `navigateTo(bot, block.position.x, block.position.y, block.position.z, 2, timeoutMs)`
    - range=2 because you need to be within reach distance, not standing on the block

    1. Create `body/dig.js` with this exact export:

       ```javascript
       // dig.js — Dig blocks with post-dig verification
       ```

       Export `digBlock(bot, block)`:
       - Save the position BEFORE dig: `const pos = block.position.clone()`
       - Save block name before: `const nameBefore = bot.blockAt(pos)?.name`
       - Call `await bot.dig(block)` (throws on diggingAborted, so wrap in try/catch)
       - After dig, check: `const nameAfter = bot.blockAt(pos)?.name`
       - If `nameAfter === nameBefore`: return `{ success: false, reason: 'block_unchanged' }` (server-side protection detected)
       - If names differ: return `{ success: true, before: nameBefore, after: nameAfter }`
       - On error (e.g., diggingAborted): return `{ success: false, reason: err.message }`

    2. Create `body/place.js` with this exact export:

       ```javascript
       // place.js — Place blocks with post-place verification
       import { Vec3 } from 'vec3'
       ```

       Export `placeBlock(bot, referenceBlock, faceVector)`:
       - Compute target position: `const targetPos = referenceBlock.position.plus(faceVector)`
       - Call `await bot.placeBlock(referenceBlock, faceVector)` (wrap in try/catch)
       - After place, check: `const placed = bot.blockAt(targetPos)`
       - If `!placed || placed.name === 'air'`: return `{ success: false, reason: 'placement_failed' }`
       - Otherwise: return `{ success: true, placedBlock: placed.name }`
       - On error: return `{ success: false, reason: err.message }`

       Also export `placeBlockOnTop(bot, baseBlock)` as a convenience:
       - Calls `placeBlock(bot, baseBlock, new Vec3(0, 1, 0))` — places on top face
       - This is the most common placement direction

       Also export the standard face vector constants:

       ```javascript
       export const FACE = {

         NORTH:  new Vec3(0, 0, -1),
         SOUTH:  new Vec3(0, 0, 1),
         EAST:   new Vec3(1, 0, 0),
         WEST:   new Vec3(-1, 0, 0),
       }
       ```
