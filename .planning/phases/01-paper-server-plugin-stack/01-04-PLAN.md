All commands run on Glass via SSH.

Pre-generation fills region files so agents don't hit lag spikes when exploring. A 2000 block radius = ~125 region files (each .mca covers 512x512 blocks).

1. Start Chunky pre-generation via server console.

   Method: Use docker exec with stdin to send commands to the Paper server console:

   ssh glass "docker exec -i mc-server bash -c 'echo \"chunky center 0 0\" > /proc/1/fd/0'"
   sleep 2
   ssh glass "docker exec -i mc-server bash -c 'echo \"chunky radius 2000\" > /proc/1/fd/0'"
   sleep 2
   ssh glass "docker exec -i mc-server bash -c 'echo \"chunky world hermescraft-world\" > /proc/1/fd/0'"
   sleep 2
   ssh glass "docker exec -i mc-server bash -c 'echo \"chunky start\" > /proc/1/fd/0'"

   Alternative if /proc/1/fd/0 doesn't work: use RCON.
   Check if RCON is enabled in server.properties:
   ssh glass "grep rcon /opt/hermescraft/server/server.properties"

   If RCON is not enabled, enable it:
   ssh glass "cat >> /opt/hermescraft/server/server.properties << 'EOF'
enable-rcon=true
rcon.port=25575
rcon.password=hermescraft
EOF"
   ssh glass "docker restart mc-server" && sleep 30

   Then use RCON to send commands:
   ssh glass "docker exec mc-server bash -c 'apt-get update -qq && apt-get install -qq -y mcrcon 2>/dev/null; mcrcon -H localhost -P 25575 -p hermescraft \"chunky center 0 0\" \"chunky radius 2000\" \"chunky world hermescraft-world\" \"chunky start\"'"

   If mcrcon is not available, install a simple RCON client:
   ssh glass "docker exec mc-server bash -c 'echo -ne \"\\x0d\\x00\\x00\\x00\\x00\\x00\\x00\\x00\\x03\\x00\\x00\\x00hermescraft\\x00\\x00\" | nc -w 3 localhost 25575'"

   SIMPLEST FALLBACK: Use docker attach to interact with the server console directly. But this blocks. Instead:
   ssh glass "docker exec mc-server bash -c 'printf \"chunky center 0 0\\nchunky radius 2000\\nchunky world hermescraft-world\\nchunky start\\n\" > /proc/1/fd/0'"

2. Monitor pre-generation progress (this takes 5-15 minutes for 2000 block radius):
   ssh glass "docker logs mc-server 2>&1 | grep -i chunky | tail -5"

   Check every 2 minutes until Chunky reports completion:
   ssh glass "docker logs mc-server 2>&1 | grep -i 'complete\|finish\|100%' | grep -i chunky"

3. Once pre-generation completes, verify region files were created:
   ssh glass "ls /opt/hermescraft/server/hermescraft-world/region/ | wc -l"

   With 2000 block radius, expect at least 50+ region files. Each region file is a 512x512 area.
   2000 block radius = diameter of 4000 blocks = ~8 regions across = ~64 region files minimum.

4. Check the region file sizes to ensure they contain actual terrain data (not empty):
   ssh glass "du -sh /opt/hermescraft/server/hermescraft-world/region/"
