   a) Timber / UltimateTimber — try Hangar API:
      ssh glass "curl -L -o /opt/hermescraft/server/plugins/UltimateTimber.jar 'https://hangar.papermc.io/api/v1/projects/UltimateTimber/versions?limit=1&platform=PAPER' 2>/dev/null"

   IMPORTANT: Hangar API for downloads is: GET /api/v1/projects/{slug}/versions/{version}/{platform}/download
   Use the Hangar website to find the latest version, then construct the API URL.

   Practical approach — use curl to query Hangar API for latest version, then download:
   ssh glass 'VERSION=$(curl -s "https://hangar.papermc.io/api/v1/projects/UltimateTimber/latestrelease" 2>/dev/null); curl -L -o /opt/hermescraft/server/plugins/UltimateTimber.jar "https://hangar.papermc.io/api/v1/projects/UltimateTimber/versions/$VERSION/PAPER/download" 2>/dev/null'

All commands run on Glass via SSH.

1. Configure LuckPerms — create a "bot" group with broad plugin command access.
   Use docker exec to run commands via the server console (RCON or direct console input):

   ssh glass "docker exec mc-server bash -c 'echo \"lp creategroup bot\" >> /proc/1/fd/0'"

   NOTE: The above approach may not work with all Docker setups. Alternative approaches:

   a) Use the Paper server console via docker exec with stdin:
      ssh glass "docker exec -i mc-server /bin/bash -c 'echo lp creategroup bot > /proc/1/fd/0'"

   b) If console injection doesn't work, edit LuckPerms config files directly:
      Create /opt/hermescraft/server/plugins/LuckPerms/json-storage/groups/bot.json:

      ssh glass 'mkdir -p /opt/hermescraft/server/plugins/LuckPerms/json-storage/groups && cat > /opt/hermescraft/server/plugins/LuckPerms/json-storage/groups/bot.json << '"'"'JSONEOF'"'"'
{
  "name": "bot",
  "permissions": [
    { "permission": "ultimatetimber.*", "value": true },
    { "permission": "veinminer.*", "value": true },
    { "permission": "essentials.*", "value": true },
    { "permission": "mcmmo.*", "value": true },
    { "permission": "quickshop.*", "value": true },
    { "permission": "blockbeacon.*", "value": true },
    { "permission": "servertap.*", "value": true },
    { "permission": "chunky.*", "value": true },
    { "permission": "skript.*", "value": true },
    { "permission": "minecraft.command.*", "value": true }
  ]
}
JSONEOF'

   c) PREFERRED APPROACH: Use RCON or screen/tmux if available, OR simply set permissions via LuckPerms YAML/JSON storage.

   Most reliable: Write the group file directly to LuckPerms storage, then restart or reload.
   LuckPerms default storage type is "h2" (embedded DB). If so, use console commands instead.

   Check storage type:
   ssh glass "grep 'storage-method' /opt/hermescraft/server/plugins/LuckPerms/config.yml 2>/dev/null"

   If storage-method is "h2" (default), must use console commands. Use this approach:
   ssh glass "docker exec mc-server rcon-cli lp creategroup bot" (if RCON is configured)

   OR configure LuckPerms to use YAML storage before proceeding:
   ssh glass "sed -i 's/storage-method: .*/storage-method: yaml/' /opt/hermescraft/server/plugins/LuckPerms/config.yml"
   Then restart, then write YAML group file.

   THE SIMPLEST PATH: Switch LuckPerms to yaml storage, restart, then create group files:

   Step A: Set LuckPerms to use yaml storage:
   ssh glass "sed -i 's/storage-method:.*/storage-method: yaml/' /opt/hermescraft/server/plugins/LuckPerms/config.yml"

   Step B: Restart server:
   ssh glass "docker restart mc-server" && sleep 30

   Step C: Create bot group YAML file:
   ssh glass 'mkdir -p /opt/hermescraft/server/plugins/LuckPerms/yaml-storage/groups && cat > /opt/hermescraft/server/plugins/LuckPerms/yaml-storage/groups/bot.yml << '"'"'EOF'"'"'
name: bot
permissions:
- ultimatetimber.*
- veinminer.*
- essentials.*
- mcmmo.*
- quickshop.*
- blockbeacon.*
- servertap.*
- chunky.*
- skript.*
- minecraft.command.*
EOF'

   Step D: Restart again to pick up the new group:
   ssh glass "docker restart mc-server" && sleep 30

2. Configure EssentialsX economy settings:
   Edit /opt/hermescraft/server/plugins/Essentials/config.yml:

   ssh glass "sed -i 's/starting-balance:.*/starting-balance: 100/' /opt/hermescraft/server/plugins/Essentials/config.yml"

   Ensure /home, /warp, /back, /pay are enabled (they should be by default in EssentialsX).
   Check that they're not in the disabled-commands list:
   ssh glass "grep -A 20 'disabled-commands:' /opt/hermescraft/server/plugins/Essentials/config.yml"
   If any of home, warp, back, pay are listed there, remove them.

3. Restart server one final time to apply all config changes:
   ssh glass "docker restart mc-server" && sleep 30

4. Verify LuckPerms loaded the bot group:
   ssh glass "docker logs mc-server 2>&1 | tail -30 | grep -i luckperms"
