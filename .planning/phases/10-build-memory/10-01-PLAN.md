    - loadBuildHistory() returns [] if file missing or JSON parse fails
    - recordBuild({ name, origin, dimensions, blockCount, builder }) appends to history array and writes to disk
    - recordBuild adds a `date` field (ISO string) automatically
    - getBuildHistoryForPrompt() returns empty string when history is empty
    - getBuildHistoryForPrompt() returns formatted string listing each build with name, location, dimensions, block count, date
    - History array capped at 20 entries (oldest removed first)

  <action>
Create `mind/build-history.js` following the same module pattern as `mind/memory.js` and `mind/locations.js`:

1. Module state: `let HISTORY_FILE = ''` and `let history = []`
2. `initBuildHistory(config)` — sets `HISTORY_FILE = join(config.dataDir, 'build_history.json')`. Ensures dataDir exists with mkdirSync recursive. This is a lightweight init (no disk read) — loading happens separately.
3. `loadBuildHistory()` — reads HISTORY_FILE if it exists, JSON.parse into `history` array. On missing file or parse error, set `history = []`. Return the array.
4. `recordBuild({ name, origin, dimensions, blockCount, builder })` — creates a record object: `{ name, origin: {x,y,z}, dimensions: {x,y,z}, blockCount, builder, date: new Date().toISOString() }`. Pushes to `history`. If `history.length > 20`, shift oldest. Writes `history` to HISTORY_FILE as JSON with 2-space indent. Silent catch on write error (console.log warning).
5. `getBuildHistoryForPrompt()` — if `history.length === 0` return `''`. Otherwise build a string:
   ```
   Previous builds:
     small_cabin at 100,64,200 (5x4x5, 47 blocks) — 2026-03-22
     watchtower at 150,64,180 (3x8x3, 38 blocks) — 2026-03-22
   ```
   Format each entry as: `  {name} at {origin.x},{origin.y},{origin.z} ({dimensions.x}x{dimensions.y}x{dimensions.z}, {blockCount} blocks) -- {date short}`
   Date short = `date.slice(0, 10)` (YYYY-MM-DD).
6. `saveBuildHistory()` — writes current history array to disk (called by periodic save).

All functions exported as named exports. Follow conventions: one-line comment header, camelCase, no default exports, 2-space indent, no-semi, single quotes.
