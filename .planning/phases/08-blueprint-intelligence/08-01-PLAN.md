  <action>
Create body/blueprints/validate.js exporting a single function:

```
export function validateBlueprint(jsonString)
```

The function takes a raw JSON string (as an LLM would produce) and returns:
- `{ valid: true, blueprint: parsedObject }` if everything checks out
- `{ valid: false, errors: string[] }` listing ALL validation failures (not just the first)

Validation checks (in order, but collect ALL errors):

1. **JSON parse** — try JSON.parse(), push "Invalid JSON: {message}" on failure and return early
2. **Required fields** — name (string), palette (object with 1+ keys), layers (array with 1+ entries). Optional: description, size, comment
3. **Size inference** — if size is missing, infer from layers[0].grid: x = grid[0].length, z = grid.length, y = layers.length. Attach inferred size to blueprint
4. **Palette block validation** — for each palette entry, check preferred[0] against mcData.blocksByName. Error: "Palette '{char}': unknown block '{name}'"
5. **Layer y ordering** — layers must be sorted ascending by y. Error: "Layer {i} y={y} is not after layer {i-1} y={prevY}"
6. **Grid dimension check** — every row in every layer must have the same length (size.x). Error: "Layer y={y} row {r}: length {len} != expected {size.x}"
7. **Grid row count** — every layer must have size.z rows. Error: "Layer y={y}: {actual} rows != expected {size.z}"
8. **Palette key check** — every non-'.'/' ' character in grid must exist in palette. Error: "Layer y={y} row {r}: unknown palette key '{char}'"

Import minecraft-data the same way build.js does:
```
import minecraftData from 'minecraft-data'
const mcData = minecraftData('1.21.1')
```

Use the project's one-line file header comment convention.
