Add a new exported function `buildDesignPrompt(description, referenceBlueprints)` to mind/prompt.js.

This function builds a DEDICATED system prompt for blueprint generation (NOT the regular game prompt). It is called only when !design is triggered, in a separate LLM call.

Parameters:
- `description` (string) — the natural language structure description from the user
- `referenceBlueprints` (array of {name, json} objects) — 3 reference blueprints selected from body/blueprints/

The function returns a system prompt string with these sections:

```
You are a Minecraft blueprint designer. Given a structure description, output ONLY valid JSON matching this exact schema:

{
  "name": "snake_case_name",
  "description": "human-readable description",
  "size": { "x": width, "y": height, "z": depth },
  "palette": {
    "CHAR": { "preferred": ["minecraft_block_name", "alt1", "alt2"], "tag": "semantic_label" }
  },
  "layers": [
    { "y": 0, "comment": "layer description", "grid": ["ROW...", ...] },
    { "y": 1, "comment": "layer description", "grid": ["ROW...", ...] }
  ]
}

Rules:
- Layers MUST be in ascending y order (y:0 = ground floor, then y:1, y:2, ...)
- '.' in grid means air/empty space
- Every non-'.' character in grid MUST be a key in palette
- Every grid row must have exactly size.x characters
- Every layer must have exactly size.z rows
- palette preferred[] must contain valid Minecraft 1.21.1 block names
- Keep structures modest: max 10x10 footprint, max 8 blocks tall
- Floor before walls before roof — build bottom-up
- Provide 2-3 material alternatives in each preferred[] list

Here are reference examples:

=== {name1} ===
{json1}

=== {name2} ===
{json2}

=== {name3} ===
{json3}

Now design: {description}

Output ONLY the JSON. No explanation, no markdown fencing, no text before or after.
```

2. Add a `designAndBuild(bot, description)` async function (exported) that:

   a. **Select 3 reference blueprints** — read all JSON files from body/blueprints/ (using the BLUEPRINTS_DIR path derived from import.meta.url), parse them, pick 3 randomly (excluding any file named `_generated.json`). Format as `{ name, json: JSON.stringify(bp, null, 2) }`.

   b. **Build the design prompt** — call `buildDesignPrompt(description, references)`.

   c. **Make a dedicated LLM call** — call `queryLLM(designPrompt, 'Generate the blueprint now.')`. This is a SEPARATE call from the normal game loop — the design prompt replaces the system prompt for this one call only. The conversation history will include this exchange, which is fine.

   d. **Extract JSON from response** — the LLM raw response should be pure JSON. Try to extract it:
      - First try: `JSON.parse(result.raw.replace(/<think>[\s\S]*?<\/think>/g, '').trim())`
      - Second try: regex extract `\{[\s\S]*\}` from the stripped text, then JSON.parse
      - If both fail: return `{ success: false, reason: 'LLM did not produce valid JSON' }`

   e. **Validate** — call `validateBlueprint(jsonString)`.
      - If invalid: return `{ success: false, reason: 'Blueprint validation failed: ' + errors.join('; ') }`
      - (Future: could retry with error feedback, but for now just fail with descriptive error)

   f. **Write to temp file** — write validated blueprint JSON to `body/blueprints/_generated.json` (overwriting any previous). This allows the existing `build()` function to load it by name.

   g. **Auto-execute build** — get bot position, call `build(bot, '_generated', Math.round(pos.x), Math.round(pos.y), Math.round(pos.z))`. Return the build result.

   h. **Return result** — `{ success: true/false, ... }` from build, or error from validation.

3. In the `think()` function, add handling for `result.command === 'design'` BEFORE the generic dispatch block (similar to how 'sethome' and 'idle' are handled):

```javascript
if (result.command === 'design') {
  const description = result.args.description || ''
  if (!description) {
    console.log('[mind] !design missing description')
    lastActionTime = Date.now()
    return
  }
  console.log('[mind] designing:', description)
  skillRunning = true
  const designResult = await designAndBuild(bot, description)
  skillRunning = false
  console.log('[mind] design result:', designResult.success ? 'OK' : designResult.reason)
  lastActionTime = Date.now()

  // Record successful builds
  if (designResult.success) {
    const pos = bot.entity.position
    addWorldKnowledge(`Designed and built "${description}" at ${Math.round(pos.x)},${Math.round(pos.y)},${Math.round(pos.z)}`)
    saveLocation(`custom_build`, Math.round(pos.x), Math.round(pos.y), Math.round(pos.z), 'build')
  }

  setTimeout(() => think(bot, { trigger: 'skill_complete', skillName: 'design', skillResult: designResult }), 0)
  return
}
```
