**Step 1: Create infra/start-models.sh locally**

Create the file `infra/start-models.sh` in the hermescraft repo with this exact content:

```bash
#!/bin/bash
# start-models.sh — Sequential dual-model startup for RunPod A6000 48GB
# Main brain: llama-server (heretic Q6_K GGUF) on port 8000
# Secondary brain: vLLM (Qwen3.5-9B) on port 8001
set -e

MODEL_DIR="${MODEL_DIR:-/models}"
MAIN_MODEL="$MODEL_DIR/qwen35-27b-heretic-q6_k.gguf"
SECONDARY_MODEL="Qwen/Qwen3.5-9B"

echo "[start-models] Starting main brain (llama-server) on port 8000..."
llama-server \
  -m "$MAIN_MODEL" \
  --port 8000 \
  --host 0.0.0.0 \
  --n-gpu-layers 999 \
  --flash-attn \
  --ctx-size 32768 \
  --chat-template qwen3 \
  --served-model-name hermes \
  -n 512 &
LLAMA_PID=$!

echo "[start-models] Waiting for llama-server health..."
until curl -sf http://localhost:8000/health > /dev/null 2>&1; do sleep 5; done
echo "[start-models] llama-server ready on port 8000 (PID $LLAMA_PID)"

# Wait 30s for CUDA allocations to settle before starting second model
sleep 30

echo "[start-models] Starting secondary brain (vLLM) on port 8001..."
vllm serve "$SECONDARY_MODEL" \
  --port 8001 \
  --host 0.0.0.0 \
  --gpu-memory-utilization 0.38 \
  --max-model-len 32768 \
  --enforce-eager \
  --served-model-name secondary &
VLLM_PID=$!

echo "[start-models] Waiting for vLLM health..."
until curl -sf http://localhost:8001/health > /dev/null 2>&1; do sleep 5; done
echo "[start-models] vLLM ready on port 8001 (PID $VLLM_PID)"

echo "[start-models] Both models running."
nvidia-smi --query-gpu=memory.used,memory.free,memory.total --format=csv,noheader

wait $LLAMA_PID $VLLM_PID
```

Note: `-n 512` (not 128) is the llama-server max output tokens flag. The actual per-request
max_tokens is controlled by the API caller (mind/llm.js). llama-server's `-n` sets the upper
bound the server will allow.

**Step 2: SSH into pod and install llama-server**

Use the expect SSH pattern. Run these commands on the pod:

```bash
# Install llama.cpp server (pre-built binary)
pip install huggingface_hub
apt-get update && apt-get install -y cmake build-essential curl
# Download pre-built llama.cpp release with CUDA support
curl -L https://github.com/ggerganov/llama.cpp/releases/latest/download/llama-server-linux-cuda12-x86_64.tar.gz | tar xz -C /usr/local/bin/
# Verify
llama-server --version
```

If pre-built binary is not available at that URL, build from source:
```bash
git clone --depth 1 https://github.com/ggerganov/llama.cpp /tmp/llama.cpp
cd /tmp/llama.cpp && cmake -B build -DGGML_CUDA=ON && cmake --build build --target llama-server -j$(nproc)
cp build/bin/llama-server /usr/local/bin/
```
